POWER AUTOMATE BUILD GUIDE

Flow 01 — RFP Creation

Step-by-step guide to build the RFP document generation flow in Power Automate. This flow receives inputs from the Copilot Studio RFP Creator agent, selects the correct Word template, populates it, saves to SharePoint, and notifies via Teams.

Prerequisites

Connector 2

Connector 3

Site with "RFP Templates" and "RFP Tracking" list

Step-by-Step Build Instructions
1

Create the Flow from Copilot Studio

This ensures the flow is automatically connected to your agent.

  1. Open Copilot Studio and navigate to your RFP Creator Agent
  2. Open the Create RFP topic
  3. Find the Call an action node (or add one with the + button)
  4. Select Create a new flow
  5. Power Automate opens with a pre-configured trigger and response action
  6. Name the flow: RFP Creation - Populate Template
Tip: Creating flows from Copilot Studio auto-registers them. You will not need to manually link the flow ID later.
2

Configure Trigger Inputs Copilot Studio Trigger

The trigger is already added. You need to define the 7 input parameters the agent will send.

  1. Click the trigger step at the top of the flow
  2. Click + Add an input for each parameter below
  3. For each input, select Text as the type and enter the name exactly as shown
Input NameTypeRequiredDescription
rfpReferenceTextYesRFP reference number (e.g., KG-RFP-2026-001)
engagementTypeTextYesTurnkey, ManagedServices, or AugmentedResources
projectNameTextYesName of the project
projectDescriptionTextNoBrief scope and objectives
budgetRangeTextNoBudget estimate (e.g., SGD 500K to 1M)
timelineTextNoExpected project timeline
submissionDeadlineTextNoPartner submission deadline
3

Initialize Variable Variable

Store the RFP reference in a variable for use throughout the flow.

  1. Click + New step
  2. Search for Initialize variable and select it
  3. Configure as follows:
FieldValue
NamerfpReference
TypeString
ValueSelect rfpReference from Dynamic content (from the trigger)
Also add a second Initialize variable right after this one for templatePath (Type: String, Value: leave empty). This will be set by the Switch action in the next step.
4

Switch on Engagement Type Control

Route to the correct Word template based on the engagement type.

  1. Click + New step and search for Switch (under Control)
  2. Set On to: select engagementType from Dynamic content
  3. Add 3 cases as shown below. In each case, add a Set variable action.
Case: Turnkey
Set templatePath to:
/RFP Templates/Turnkey Project/rfp-template.docx
Case: ManagedServices
Set templatePath to:
/RFP Templates/Managed Services/rfp-template.docx
Case: AugmentedResources
Set templatePath to:
/RFP Templates/Augmented Resources/rfp-template.docx
Important: The case values must match exactly what the Copilot Studio topic sends: Turnkey, ManagedServices, AugmentedResources. These are case-sensitive.
5

Populate a Word Template Word Online (Business)

Fill in the RFP Word template with the collected project data.

  1. Click + New step and search for Populate a Microsoft Word template
  2. Select the Word Online (Business) connector action
  3. Configure the location and file:
FieldValue
LocationYour SharePoint site (e.g., Kuok Group IT - RFP)
Document LibrarySelect your document library
FileSelect templatePath variable from Dynamic content

Then map each template placeholder to the trigger inputs:

Template PlaceholderDynamic Content Value
rfpReferencerfpReference (variable)
projectNameprojectName (trigger)
projectDescriptionprojectDescription (trigger)
budgetRangebudgetRange (trigger)
timelinetimeline (trigger)
submissionDeadlinesubmissionDeadline (trigger)
Prerequisite: Your Word template (.docx) files must exist in SharePoint at the paths defined in Step 4. Each template should contain content controls with names matching the placeholders above (e.g., a Plain Text content control named rfpReference).
6

Create File in SharePoint SharePoint

Save the populated document to a dedicated folder in SharePoint.

  1. Click + New step and search for Create file (SharePoint)
  2. Configure:
FieldValue
Site AddressYour SharePoint site URL
Folder Path/Issued RFPs/ then insert rfpReference variable
File NameInsert rfpReference variable, then type -RFP.docx
File ContentSelect Microsoft Word Document from the Populate Word Template outputs (Dynamic content)
Result: A file like /Issued RFPs/KG-RFP-2026-001/KG-RFP-2026-001-RFP.docx will be created in SharePoint.
7

Create Tracking Item SharePoint

Log the new RFP in a SharePoint tracking list for status monitoring.

  1. Click + New step and search for Create item (SharePoint)
  2. Configure:
FieldValue
Site AddressYour SharePoint site URL
List NameRFP Tracking
TitlerfpReference (variable)
ProjectNameprojectName (trigger)
EngagementTypeengagementType (trigger)
StatusDraft (type this as text)
SubmissionDeadlinesubmissionDeadline (trigger)
Prerequisite: Create a SharePoint list called RFP Tracking with columns: Title (single line), ProjectName (single line), EngagementType (choice: Turnkey/ManagedServices/AugmentedResources), Status (choice: Draft/Issued/Collecting/Evaluating/Complete), SubmissionDeadline (date).
8

Post Teams Notification Microsoft Teams

Notify the Deal Room channel that a new RFP has been created.

  1. Click + New step and search for Post message in a chat or channel
  2. Configure:
FieldValue
Post asFlow bot
Post inChannel
TeamSelect your RFP team (e.g., Kuok IT RFP Team)
ChannelSelect the Deal Room channel
MessageSee message template below
Message template:
New RFP Created: [rfpReference]
Project: [projectName]
Type: [engagementType]
Deadline: [submissionDeadline]

Use Dynamic content to insert the values from the trigger and variables. You can format this with HTML if you toggle the rich text editor.
9

Respond to Copilot Studio Response

Send the results back to the agent so it can display the RFP reference and document link.

  1. The Respond to Copilot action was auto-created when you started the flow from Copilot Studio. Click on it at the bottom.
  2. Click + Add an output twice to create two outputs:
Output NameTypeValue (Dynamic Content)
rfpReferenceTextrfpReference (variable)
documentUrlTextPath (from Create File step output)
Done! These outputs map back to the Copilot Studio topic variables Topic.RFPReference and Topic.DocumentUrl, which the agent displays in its confirmation message.
Save and Test

Final Steps

  1. Save the flow in Power Automate
  2. Return to Copilot Studio — the flow should appear in the action node dropdown
  3. In the Create RFP topic, select the flow and map the topic variables to the flow inputs/outputs
  4. Test by opening the agent in the Copilot Studio test pane and saying "Create an RFP"
  5. Verify the document appears in SharePoint and the Teams notification posts
Common Issues:
• Word template not found — Ensure the .docx files exist at the exact paths in SharePoint
• Missing columns — Ensure the RFP Tracking list has all required columns
• Connection errors — Sign in to all three connectors (SharePoint, Word Online, Teams) when prompted
• Dynamic content not showing — Ensure previous steps are connected (check the "runs after" setting)
